# 앱 내장 브라우저로 진행하는 상세 가이드 (Claude 코파일럿 방식)

> [디멘드젠-셋업-가이드.md](디멘드젠-셋업-가이드.md)의 **Part 1(수동 웹 작업)** 을, Claude Code 앱의
> **내장 브라우저**를 띄워놓고 실시간으로 함께 진행하는 방식의 상세 실행서입니다.
> 핵심은 **역할 분담**: Claude는 "열고·읽고·짚어주고·검증", 사용자는 "로그인·캡차·카드·최종 동의".

---

## 0. 이 방식이 어떻게 작동하나

### 역할 분담 (반드시 이해하고 시작)

| 구분 | Claude가 하는 일 | 사용자가 직접 하는 일 |
|------|------------------|----------------------|
| 페이지 이동 | 정확한 URL로 브라우저 열기 (`navigate`) | — |
| 화면 파악 | 페이지 읽어서 지금 뭐가 보이는지 파악 (`read_page`), 필요 시 스크린샷 | 화면 눈으로 확인 |
| 안내 | "지금 이 버튼(파란 '만들기') 누르세요" 처럼 실시간 지목 | 지목된 곳 클릭 |
| 비민감 입력 | 계정 이름 등 안전한 값 대신 입력 가능 (`form_input`) | — |
| **로그인** | ✋ 안 함 (비밀번호 입력 금지) | **구글 아이디·비밀번호·2단계 인증 직접 입력** |
| **reCAPTCHA** | ✋ 안 함 (봇 우회 금지) | **"로봇이 아닙니다"·그림 퀴즈 직접 해결** |
| **카드/결제** | ✋ 안 함 (결제정보 입력 금지) | **카드번호 등 직접 입력** |
| **최종 동의/제출** | ✋ 대신 안 누름 | **OAuth "허용", 약관 "제출" 직접 클릭** |
| 검증 | 결과값(MCC ID 10자리, 토큰 22자 등) 읽어서 맞는지 확인 → `.env` 반영 | — |

### 앱 내장 브라우저의 특성 (미리 알아둘 것)
- 내 실제 크롬과 **별개의 브라우저**입니다 → **구글 로그인을 새로 1회** 해야 합니다.
- 브라우저 창은 앱 화면에 함께 뜨고, **사용자도 그 창에 직접 클릭·타이핑**할 수 있습니다.
  로그인·캡차·카드 같은 민감 입력은 이 방식으로 **사용자가 그 창에 직접** 칩니다.
- 만약 내장 브라우저 창에 직접 타이핑이 안 되는 환경이면 → **로그인 게이트가 걸리는 구간만
  본인 크롬에서** 처리하고 값(토큰 등)을 가져오는 하이브리드로 전환합니다. (아래 각 단계의 "대안" 참고)

### 보안 원칙 (녹화 필수 수칙)
- 비밀번호·카드번호·토큰이 **화면에 뜨는 구간에서는 스크린샷을 찍지 않습니다.** 제가 알아서 피합니다.
- 그 구간은 녹화 시 **모자이크/편집** 하세요. (마지막 "녹화 체크포인트" 참고)

## 시작 전 준비물
- [ ] **새 구글 계정** 아이디/비밀번호 (2단계 인증 쓰면 폰도 옆에)
- [ ] 카드 1장 (GCP 결제 등록용)

준비되면 저에게 **"앱 내장 브라우저로 셋업 시작하자"** 라고 하면 아래 순서대로 진행합니다.

## 단계별 상세 진행

각 단계는 **[내 동작] → [당신 동작] → [검증 체크포인트]** 순서입니다.

### STEP 0. 브라우저 세션 열고 구글 로그인

- **[내 동작]** 내장 브라우저를 `https://accounts.google.com` 으로 엽니다. 화면을 읽어 로그인
  폼이 떴는지 확인하고 알려드립니다.
- **[당신 동작]** 브라우저 창에 **새 구글 계정으로 직접 로그인** (아이디 → 비밀번호 → 2단계 인증).
  ※ 저는 비밀번호를 입력하지 않습니다.
- **[검증]** 로그인 후 제가 페이지를 읽어 **우측 상단에 계정 아바타/이메일**이 보이는지 확인 →
  "로그인 확인됨, 이 계정으로 끝까지 진행합니다" 라고 짚어드립니다.
- **[대안]** 창에 직접 타이핑이 안 되면 → 본인 크롬에서 이 계정으로 로그인해 두고, 이후 단계는
  값만 제게 알려주는 방식으로 진행.

> ⚠️ 여기서 로그인한 계정이 **끝까지 동일**해야 합니다. 이후 어느 단계에서도 계정이 바뀌면
> `USER_PERMISSION_DENIED`·"액세스 차단됨"이 납니다.

### STEP 1. 관리자(MCC) 계정 만들기

> **왜 MCC부터?** 개발자 토큰(STEP 2)은 **관리자(MCC) 계정에서만** 발급됩니다. 일반 광고계정만으론
> API 토큰을 못 받습니다. 그래서 제일 먼저 MCC를 만듭니다.
>
> **MCC ≠ 광고계정.** 관리자 계정은 "계정들을 관리하는 상위 계정"이라 **그 자체로는 광고를 못
> 돌립니다.** 실제 디멘드젠 광고는 MCC **아래의 광고계정**에서 집행됩니다. → MCC를 만든 뒤
> **STEP 1.5에서 광고계정을 하나 붙입니다.**

#### 1-A. 시작 전 확인 (이메일 중복 함정)

- **[내 동작]** 브라우저를 `https://ads.google.com/home/tools/manager-accounts` 로 엽니다.
  > 💡 이 **전용 URL**로 들어가는 게 중요합니다. 그냥 `ads.google.com`으로 가면 일반 광고
  > **가입(스마트 모드) 화면**으로 새서, 관리자 계정이 아니라 캠페인부터 만들라고 유도합니다.
- **[내 동작]** 화면을 읽어 지금 상태를 알려드립니다. 두 경우로 갈립니다:
  - **"관리자 계정 만들기" 버튼이 보이면** → 1-B로 진행.
  - **"이미 이 이메일로 계정이 있습니다" / 기존 광고계정 대시보드가 뜨면** → 이 구글 계정은 이미
    일반 광고계정에 묶여 있는 겁니다. 그래도 **관리자 계정은 별도로 만들 수 있습니다**(같은 이메일
    가능). 제가 "관리자 계정 만들기" 링크를 다시 찾아 짚어드립니다. 정 꼬이면 **광고 전용 새 Gmail**로
    STEP 0부터 다시 하는 게 가장 깔끔합니다.

#### 1-B. 관리자 계정 생성 폼 채우기

- **[당신 동작]**
  1. 파란 **"관리자 계정 만들기"** 클릭
  2. **계정 이름** 입력 (예: `내이름_광고관리`)
     - 내부 표시용 이름이라 **나중에 변경 가능**합니다. 부담 없이. (제가 대신 입력 가능)
  3. **"이 계정을 어떻게 사용하시겠어요?"**
     - **"다른 사람의 계정 관리"** = 여러 광고주/계정을 대행·관리 (에이전시형)
     - **"내 계정 관리"** = 내 광고계정들만 묶어서 관리
     - → **모르면 "다른 사람의 계정 관리"** 선택. 나중에 광고계정을 새로 붙이는 데 제약이 없습니다.
  4. ★★ **국가=대한민국 / 시간대=(GMT+09:00) 서울 / 통화=KRW**
     - ⚠️ **시간대와 통화는 계정 생성 후 절대 변경 불가**입니다. (계정 이름만 바꿀 수 있음)
     - → 제가 화면을 읽어 **시간대가 정확히 '서울'인지, 통화가 'KRW(원)'인지 실시간으로 확인**하고,
       미국(로스앤젤레스)·USD 등으로 잘못 잡혀 있으면 **제출 전에** 바로 잡아드립니다.
       (실전에서 시간대가 미국으로 잡혀 예산 리셋·리포트가 전부 미국 기준이 된 사례가 있었음.)
  5. **"로봇이 아닙니다"** 체크 → 그림 퀴즈가 나오면 풀기 → **반드시 당신이 직접**
     (제가 캡차를 대신 풀지 않습니다)
  6. **제출** / **"계정 만들기"** 클릭 → **당신이 직접**
  7. 완료 화면에서 **"계정 탐색하기"**(또는 "계정으로 이동") 클릭

- **[검증]** 제가 페이지 상단(계정 이름 옆/우측 상단)에서 **10자리 계정 ID(예: 123-456-7890)** 를
  읽어드립니다. 형식이 맞으면 **"MCC ID = XXX-XXX-XXXX 확보"** 로 확정하고, 하이픈을 뺀 숫자
  10자리를 `.env`의 `GOOGLE_ADS_LOGIN_CUSTOMER_ID` 후보로 메모합니다.
  > 💡 **나중에 MCC ID 다시 찾기**: 관리자 계정 로그인 후 화면 **우측 상단**에 항상 10자리로 표시됩니다.

#### 1-C. (참고) 이메일 인증

- 계정 생성 후 구글이 **확인 이메일**을 보낼 수 있습니다. 받은편지함에서 인증 링크를 눌러두면
  이후 단계에서 계정 제한을 피할 수 있습니다. (필수는 아니지만 권장)

#### STEP 1.5. MCC 아래 "광고계정" 준비 (실제 광고가 나갈 곳)

MCC는 관리용 껍데기라 광고를 직접 못 돌립니다. 디멘드젠 광고가 실제로 집행되고 **결제(카드)가
연결될 광고계정**이 하나 필요합니다. 두 가지 방법 중 하나:

- **(권장) MCC 아래에 새 광고계정 생성**
  - **[내 동작]** 관리자 계정 화면에서 **"계정" → "+" (새 계정 만들기)** 위치를 짚어드립니다.
  - **[당신 동작]** 새 광고계정 이름 + **국가=대한민국 / 시간대=서울 / 통화=KRW**(여기서도 동일하게
    확인) 입력 후 생성. → 결제(카드) 연결은 [디멘드젠-셋업-가이드.md](디멘드젠-셋업-가이드.md)의
    Part 1 - 3단계에서 진행합니다.
  - **[검증]** 새 광고계정의 **10자리 ID**를 읽어 `.env`의 `GOOGLE_ADS_CUSTOMER_ID`(기본 광고계정)로 메모.

- **(대안) 이미 있는 광고계정을 MCC에 연결**
  - 기존 광고계정이 있으면 MCC에서 **초대 → 광고계정 쪽에서 수락** 흐름으로 붙입니다. 이 연결은
    `google-ads-demandgen-setup` 스킬의 `link_account.py`로 자동화할 수 있습니다:
    ```bash
    python scripts/link_account.py --manager <MCC_ID> --client <광고계정_ID>
    python scripts/link_account.py --check --manager <MCC_ID> --client <광고계정_ID>   # 상태 확인
    ```
  - ⚠️ 개발자 토큰이 **Test 등급이면 API 자동 수락이 막힙니다**. 그땐 스크립트가 안내하는 대로
    **광고계정 → 설정 → 액세스 및 보안 → 관리자 탭에서 직접 수락**하면 됩니다(등급 무관하게 됨).

  > 💡 **꼭 MCC에 연결해야 하나?** 데이터 **조회(읽기)** 만이면 광고계정에 **직접 접근**으로도 됩니다
  > (LOGIN_CUSTOMER_ID 비워둠). 실제 실습에선 광고계정을 MCC에 안 붙이고 직접 접근해서 쓴 적도
  > 있습니다. 다만 **중앙 관리·MCC 경유 조회**를 원하면 연결하고 `LOGIN_CUSTOMER_ID`를 MCC로 설정.

- **[MCC 생성 트러블슈팅]**

  | 증상 | 원인 | 해결 |
  |------|------|------|
  | 관리자 만들기 화면이 아니라 캠페인 만들기가 뜸 | `ads.google.com` 일반 가입으로 샘 | 전용 URL `.../tools/manager-accounts` 로 다시 진입 |
  | "이 이메일은 이미 사용 중" | 이 Gmail에 이미 광고계정 존재 | 같은 이메일로도 관리자 계정 별도 생성 가능. 꼬이면 **광고 전용 새 Gmail**로 |
  | 시간대/통화를 잘못 만듦 | 생성 시 잘못 선택 (변경 불가) | 계정을 다시 만드는 수밖에 없음 → **생성 전 제가 반드시 확인** |
  | 계정이 "정지/검토 중" | 신규 계정 자동 검토 | 이메일 인증 완료 + 잠시 대기. 결제 연결 후 풀리는 경우 많음 |

### STEP 2. 개발자 토큰 신청

- **[내 동작]** 같은 계정 로그인 상태에서 브라우저를 `https://ads.google.com/aw/apicenter` 로 이동.
  **"토큰 신청(Apply for access)"** 위치를 짚고, 양식 각 칸에 뭘 넣는지 안내합니다. 웹사이트 URL,
  사용 목적("데이터 분석 및 광고 자동화") 같은 **비민감 텍스트는 제가 대신 입력**할 수 있습니다.
- **[당신 동작]** 약관 동의 체크 + **최종 "제출"** 클릭.
- **[검증]** 제가 화면의 **22자리 개발자 토큰**을 읽어 형식(22자)만 확인합니다.
  ※ 토큰 값 자체가 화면에 뜨는 구간이라 **스크린샷은 찍지 않습니다.** 값은 제가 바로 `.env`의
  `GOOGLE_ADS_DEVELOPER_TOKEN` 에 반영합니다.
- **[등급 안내]** 처음엔 **Test(Explorer) 등급**입니다. 디멘드젠 **광고 생성**까지 하려면 같은
  페이지에서 **Basic 액세스**를 신청해야 하고 **승인에 1~3영업일** 걸립니다. → 지금 신청만 걸어두고,
  승인 대기 동안 STEP 3~4를 진행하는 걸 추천합니다.

### STEP 3. GCP 결제 계정 (카드) — 없을 때만

> 이 단계에서 **GCP 프로젝트 생성·API 활성화는 브라우저가 아니라 터미널 스크립트(자동)** 가 합니다.
> 사람이 손댈 건 결제 계정이 아예 없을 때 **카드 등록**뿐입니다.

- **[내 동작]** 결제 계정 유무를 먼저 확인하고(없을 때만 진행), 브라우저를
  `https://console.cloud.google.com/billing` 로 이동해 **"결제 계정 만들기"** 를 짚어드립니다.
- **[당신 동작]** 국가=대한민국/통화=KRW, 이름·주소, **카드번호 직접 입력** → 저는 결제정보를
  입력하지 않습니다.
- **[검증]** 결제 계정이 목록에 생겼는지 화면으로 확인. 이후 프로젝트 연결은 스크립트가 자동 처리.

### STEP 4. OAuth 클라이언트 JSON 만들기

- **[내 동작]** 브라우저를 `https://console.cloud.google.com/apis/credentials` 로 이동. 상단에
  **내 프로젝트가 선택돼 있는지** 먼저 읽어 확인합니다. OAuth 동의 화면 → 클라이언트 생성 순서를
  화면 기준으로 하나씩 짚습니다.
- **[당신 동작]**
  1. (처음이면) **OAuth 동의 화면** → User type **External(외부)** → 만들기
  2. 앱 이름/이메일 입력(비민감이라 제가 대신 채워드릴 수 있음) → 저장·계속
  3. ★ **테스트 사용자(Test users)에 본인 구글 이메일 추가**
     → 제가 화면을 읽어 **이메일이 실제로 추가됐는지 확인**해드립니다 (이거 빠지면 나중에
     로그인 때 "액세스 차단됨(403)" — 최다 실수).
  4. **사용자 인증 정보 → + 만들기 → OAuth 클라이언트 ID**
  5. 애플리케이션 유형 **데스크톱 앱** 선택 → 만들기
  6. **JSON 다운로드** → **당신이 직접** (다운로드 받은 파일 경로를 제게 알려주세요)
- **[검증]** 알려주신 경로의 JSON을 `~/.google-ads-mcp/google_ads_oauth_client.json` 으로 복사하고,
  `client_id`/`client_secret` 이 들어있는지 확인합니다.

### STEP 5~9. 여기서부터 자동 (브라우저 거의 안 씀)

이제 손 웹작업은 끝. 터미널 스크립트가 이어받습니다. 제가 자동 실행합니다:

```bash
python scripts/preflight.py      # 준비 상태 진단
python scripts/bootstrap.py --yes   # 의존성→OAuth→배포→등록→검증
python scripts/verify_demandgen.py  # 디멘드젠 준비 완료 확인
```

- **딱 한 번 브라우저가 다시 등장**: `bootstrap.py`의 OAuth 동의 단계. 동의 URL이 뜨면 제가 그
  주소를 내장 브라우저로 열어드리고, **당신이 본인 계정으로 "허용"** 클릭 (STEP 4에서 test user로
  넣은 그 계정). 이후 리프레시 토큰이 자동 저장됩니다.
- 완료되면 **MCP URL + 베어러 토큰**이 `~/.google-ads-mcp/connection.json` 에 저장되고,
  `verify_demandgen.py`가 **"디멘드젠 광고 설정 준비 완료 ✅"** 를 띄웁니다.

## 내장 브라우저 vs 실제 크롬 — 갈아타야 하는 순간

이 방식은 내장 브라우저 기본이지만, 아래 경우엔 **그 구간만** 본인 크롬으로 처리하는 게 편할 수 있습니다:
- 내장 브라우저 창에 **직접 타이핑이 안 될 때** → 로그인/카드 입력 구간
- 2단계 인증이 내장 브라우저에서 막힐 때
- 이미 실제 크롬에 그 계정으로 로그인돼 있어 재로그인이 번거로울 때

갈아타더라도 **계정은 반드시 동일**해야 합니다. 값(MCC ID·토큰·JSON 경로)만 제게 넘겨주면 이어서 진행합니다.

## 실전 검증 결과 (2026-07 실제 진행에서 확인)

내장 브라우저로 어디까지 되고 어디서 막히는지, 실제로 처음부터 끝까지 돌려본 결과입니다:

| 단계 | 내장 브라우저 | 비고 |
|------|:---:|------|
| STEP 0 구글 로그인 | ⭕ | 사용자가 창에 직접 타이핑 가능. 로그인됨 |
| reCAPTCHA(계정·토큰 생성) | ⭕ | 사용자가 직접 그림 퀴즈 해결 |
| STEP 1 MCC 생성 폼 | ⭕ | 국가/시간대/통화가 이미 서울·KRW로 기본 설정됨. 이름만 채우고 제출 |
| STEP 2 개발자 토큰 신청 | ⭕ | 폼 입력·제출·토큰 확인 모두 진행됨 |
| STEP 3 GCP 결제 가입 | ⭕ | 카드 입력만 사용자 직접. 단 **활성화에 최대 24h**(open:false) |
| STEP 4 OAuth 동의(인증 플랫폼 마법사) | ⭕ | 앱정보·대상·연락처는 자동, 정책 동의만 사용자 |
| STEP 4 테스트 사용자 추가 | ⭕ | 정상 |
| STEP 4 OAuth 클라이언트 "만들기" 버튼 | ⚠️ | 자동 클릭이 안 먹힘 → **사용자가 직접 "만들기" + JSON 다운로드** |
| STEP 5 OAuth 동의(리프레시 토큰) | ⭕ | 동의 URL을 내장 브라우저로 열어 "허용", localhost 리다이렉트 정상 캡처 |
| gcloud 계정 로그인 | 🖥️ | 내장 브라우저 아님 — **시스템 기본 브라우저**가 열림. 거기서 완료 |
| STEP 1.5 광고계정 생성 "본인 인증" | ❌ | 인증 팝업이 안 열려 "다시 시도" 반복 → **실제 크롬 권장** |

**요약**: 로그인·MCC·토큰·GCP·OAuth·리프레시 토큰까지는 내장 브라우저로 충분히 진행됨.
막히는 두 곳 — **① OAuth 클라이언트 "만들기" 버튼(사용자가 직접 클릭)**,
**② 광고계정 생성의 본인 인증(실제 크롬으로)**. gcloud 인증은 시스템 브라우저로 열림.

## 한계 (솔직하게)

- **reCAPTCHA·로그인·카드·최종 동의는 자동화 불가** — 이 방식으로도 그 4가지는 당신 몫입니다.
  내장 브라우저의 가치는 "대신 클릭"이 아니라 **정확한 페이지로 데려가고, 화면을 읽어 실시간으로
  짚어주고, 결과를 검증**하는 데 있습니다.
- 구글 UI는 자주 바뀝니다 → 문서의 버튼 이름과 화면이 다를 수 있는데, 제가 **그때그때 실제 화면을
  읽어** 안내하므로 문서만 보는 것보다 안전합니다.
- 내장 브라우저 로그인 세션은 오래 유지되지 않을 수 있어, 중간에 재로그인을 요청할 수 있습니다.
