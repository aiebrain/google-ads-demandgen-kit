# 완전 초보자용 셋업 매뉴얼 (사람이 직접 하는 부분)

`guided_setup.py`가 화면에서 그대로 안내하는 내용을 문서로도 정리한 것. **자동화가 불가능한 웹 작업**
(관리자 계정 생성·개발자 토큰 신청·결제 계정·OAuth 클라이언트)을 클릭 단위로 설명한다. GCP 프로젝트
생성/API 활성화 같은 자동 가능한 부분은 스크립트가 알아서 하므로 여기 없다.

> 팁: 처음부터 끝까지 **같은 구글 계정 하나**로 진행하세요. 중간에 계정이 바뀌면 권한 문제가 생깁니다.

---

## 0. 구글 계정(이메일) 준비  — 가장 먼저

모든 것의 출발점은 **구글 계정(Gmail) 하나**다.

1. 이미 Gmail 이 있으면 → 그 계정 하나로 아래 1~4번을 전부 진행한다.
2. 없으면 → https://accounts.google.com/signup 에서 새로 만든다. (광고 전용 계정을 새로 파도 좋다.)
3. **일반 @gmail.com 이면 충분**하다(회사 도메인 Workspace 계정 불필요 — 오히려 서비스계정 방식은
   Workspace가 있어야 하므로, 개인이면 이 스킬의 OAuth 방식이 정답).
4. ★**규칙: MCC·개발자토큰·GCP 프로젝트·결제·OAuth 동의까지 전부 "같은 계정" 하나로.**
   계정을 섞으면 `USER_PERMISSION_DENIED`·로그인 차단 등 권한 오류가 난다(실제로 가장 흔한 원인).

## 1. 관리자(MCC) 계정 만들기  — 자동화 불가(로봇 방지 reCAPTCHA)

1. 주소창에 입력: https://ads.google.com/home/tools/manager-accounts
2. 구글 계정으로 로그인.
3. 파란색 **"관리자 계정 만들기"** 클릭.
4. **계정 이름**: 알아볼 이름(예: `홍길동_광고관리`).
5. **용도 선택**: 잘 모르면 **"다른 사람의 계정 관리"** 추천.
6. **국가=대한민국 / 시간대=(GMT+09:00) 서울 / 통화=KRW**. ※ 통화·시간대는 나중에 못 바꿈.
7. **"로봇이 아닙니다"** 체크(그림 퀴즈 나오면 풀기).
8. **제출** → **"계정 탐색하기"** 클릭.
9. 화면 상단/오른쪽 위의 **10자리 번호(예: 123-456-7890)** 가 MCC ID. 복사해서 스크립트에 붙여넣기.

## 2. 개발자 토큰 신청  — 자동화 불가(양식 + Google 심사 1~3일)

1. 위 **관리자 계정으로 로그인된 상태**에서 주소창에 입력: https://ads.google.com/aw/apicenter
2. **"토큰 신청"(Apply for access)** 클릭.
3. 양식: 이름/회사, 웹사이트(없으면 대표 URL), 사용 목적("데이터 분석 및 광고 자동화").
4. 약관 동의 → **제출**.
5. 화면의 **22자리 개발자 토큰** 복사. ※ 대문자 `I`(아이)/소문자 `l`(엘) 혼동 주의.
6. 처음엔 **Test(Explorer) 등급**. 실제 광고 '생성'까지 하려면 같은 페이지에서 **Basic 액세스** 신청(승인 1~3영업일). 데이터 '조회'만이면 Test로도 가능.

## 3. (자동) GCP 프로젝트 + Google Ads API 활성화

이 단계는 `guided_setup.py`가 gcloud로 **자동 처리**한다(프로젝트 생성·결제 연결·API 활성화). 사람이 직접
할 건 결제 계정이 아예 없을 때 **카드 등록**뿐(아래 3-1).

### 3-1. 결제 계정이 없을 때만 — 결제 계정 만들기
1. https://console.cloud.google.com/billing → **"결제 계정 만들기"**.
2. 국가=대한민국, 통화=KRW → 개인/사업자, 이름·주소.
3. 신용/체크카드 등록(신규는 보통 무료 크레딧, Cloud Run은 실제 청구 거의 없음).
4. 만든 결제 계정을 프로젝트에 연결 → 스크립트 다시 실행.

## 4. OAuth 클라이언트 JSON 만들기  — 반자동(콘솔 생성은 수동)

1. 주소창에 입력: https://console.cloud.google.com/apis/credentials (상단에 내 프로젝트 선택 확인)
2. 처음이면 **OAuth 동의 화면** 먼저:
   - User type = **External(외부)** → 만들기
   - 앱 이름/이메일만 채우고 저장·계속
   - ★**테스트 사용자(Test users)에 본인 구글 이메일 추가** — 안 하면 로그인 때 **"액세스 차단됨(403)"** 발생(가장 흔한 실수).
3. **사용자 인증 정보 → + 만들기 → OAuth 클라이언트 ID**.
4. 애플리케이션 유형 = **데스크톱 앱** → 이름 아무거나 → 만들기.
5. **JSON 다운로드**.
6. 받은 파일 경로를 스크립트에 붙여넣기(윈도우: 파일 Shift+우클릭 → '경로로 복사').

## 이후 (스크립트가 자동)
`python bootstrap.py` → OAuth 동의(브라우저 1회) → Cloud Run 배포 → Claude Code 등록 → 디멘드젠 검증.
막히면 `references/troubleshooting.md` 또는 스크립트의 자동 진단이 원인·해결을 알려준다.
