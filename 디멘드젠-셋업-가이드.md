# 구글애즈 디멘드젠 광고 셋업 완전 가이드 (초보자용)

> **이 문서 하나로**, 아무것도 없는 상태에서 Claude Code로 구글애즈 디멘드젠(Demand Gen) 광고를
> 만들 수 있는 환경까지 만듭니다. 완전 초보자 기준으로 **클릭 단위**까지 적었습니다. 녹화하며
> 따라 하기 좋게 "당신이 직접 하는 것"과 "Claude가 자동으로 하는 것"을 명확히 나눴습니다.

---

## 0. 먼저 큰 그림

우리가 만드는 건 이 연결입니다:

```
Claude Code (내 PC)
   │  "디멘드젠 광고 만들어줘" 같은 대화
   ▼
Cloud Run 위의 MCP 서버 (내가 배포한 나만의 서버)
   │  Authorization: Bearer <토큰>
   ▼
Google Ads API  →  실제 광고 계정
```

시중의 무료 구글애즈 연동 서버는 대부분 **읽기 전용**이라 캠페인을 못 만듭니다. 그래서 **쓰기(생성·수정)가
되는 나만의 서버**를 클라우드에 한 번 올려두고, 어느 PC에서든 같은 주소로 씁니다.

### 소요 시간 / 비용
- **손으로 하는 웹 작업**: 약 40~60분 (계정·토큰·OAuth 준비)
- **Claude 자동 작업**: 약 10분 (배포·등록·검증)
- **두 개의 외부 대기(중요)**:
  - **GCP 결제 활성화**: 최대 **24시간** (이게 되어야 Cloud Run 배포 가능)
  - **개발자 토큰 Basic 승인**: 최대 **5영업일** (이게 되어야 실계정 광고 생성 가능) — 설계문서·프로젝트ID 필요
  → 즉 **"셋업 당일 = 광고 실행"은 불가능**합니다. 당일엔 자격증명 검증까지, 광고는 승인 후.
- **비용**: Cloud Run은 무료 한도가 넉넉해 실제 청구는 거의 없음. 단, GCP **결제 계정(카드) 등록은 필수**.

### 준비물 체크리스트
- [ ] 구글 계정(Gmail) 1개 — **처음부터 끝까지 이 계정 하나로만** 진행
- [ ] 신용/체크카드 1장 (GCP 결제 등록용, 실제 청구는 거의 없음)
- [ ] 이 PC에 Python 3.10 이상
- [ ] Claude Code 설치됨
- [ ] 두 개의 스킬 설치: `google-ads-direct-mcp`, `google-ads-demandgen-setup`

> ⚠️ **가장 흔한 실패 원인**: 중간에 구글 계정이 바뀌는 것. MCC·개발자 토큰·GCP 프로젝트·결제·OAuth
> 동의까지 **전부 같은 계정 하나로** 하세요. 섞으면 `USER_PERMISSION_DENIED`, "액세스 차단됨" 같은
> 권한 오류가 납니다.

## Part 1 — 당신이 직접 하는 웹 작업 (자동화 불가)

이 4가지는 로봇 방지(reCAPTCHA)와 구글 심사 때문에 **사람이 직접** 해야 합니다. 녹화 시 화면을
공유하며 천천히 진행하세요.

### 1단계. 관리자(MCC) 계정 만들기

개발자 토큰은 **관리자(MCC) 계정에서만** 발급됩니다. 일반 광고 계정만으론 못 받습니다.

1. 주소창에 입력 → https://ads.google.com/home/tools/manager-accounts
2. 구글 계정으로 로그인
3. 파란 **"관리자 계정 만들기"** 클릭
4. **계정 이름**: 알아볼 이름 (예: `내이름_광고관리`)
5. **용도**: 잘 모르면 "다른 사람의 계정 관리" 선택
6. **국가=대한민국 / 시간대=(GMT+09:00) 서울 / 통화=KRW**
   > ⚠️ **시간대·통화는 나중에 절대 못 바꿉니다.** 반드시 여기서 서울/KRW로 맞추세요. (실전에서
   > 시간대가 미국(LA)으로 잘못 잡히면 예산 리셋·리포트 날짜가 전부 미국 기준으로 계산됩니다.)
7. "로봇이 아닙니다" 체크 (그림 퀴즈 나오면 풀기)
8. **제출** → **"계정 탐색하기"**
9. 화면 상단의 **10자리 번호**(예: `123-456-7890`) = **MCC ID**. 복사해 두세요.

✅ **완료 기준**: MCC 계정 로그인됨 + 10자리 MCC ID 확보.

### 2단계. 개발자 토큰 신청

1. **위 관리자 계정으로 로그인된 상태**에서 → https://ads.google.com/aw/apicenter
2. **"토큰 신청"(Apply for access)** 클릭
3. 양식 작성: 이름/회사, 웹사이트(없으면 대표 URL), 사용 목적("데이터 분석 및 광고 자동화")
4. 약관 동의 → **제출**
5. 화면의 **22자리 개발자 토큰** 복사
   > ⚠️ 대문자 `I`(아이)와 소문자 `l`(엘) 혼동 주의.
6. 처음엔 보통 **Test(Explorer) 등급**입니다.
   - **데이터 조회만** 하려면 → Test 등급으로도 가능
   - **실제 광고 생성**까지 하려면 → **Basic 액세스** 신청이 별도로 필요

> ⚠️ **실전 확인 — Basic 액세스 신청은 생각보다 무겁습니다** (2026-07 기준):
> API Center의 "액세스 수준 → 기본 액세스 신청"을 누르면 **별도 신청서 폼**(support.google.com의
> "Google Ads API Basic Access Application", **영어 폼**)으로 넘어갑니다. 이 폼은 다음이 **필수**입니다:
> - **GCP 프로젝트 ID** (→ 아래 3단계에서 먼저 만들어야 함. 순서상 토큰 발급 후 바로 신청 불가)
> - **도구 설계 문서**(.pdf/.doc/.rtf) 업로드 — 도구의 설계·목적을 설명하는 문서
> - 비즈니스 모델·도구·대상 상세 설명, 회사 웹사이트(작동하는 URL이어야 함)
> - 심사 기간: 안내문상 **최대 5영업일**(typically within five business days)
>
> 즉 **토큰 발급(Test)은 즉시**지만, **실계정 광고 생성까지는 GCP 프로젝트 생성 → Basic 신청서 제출 →
> 최대 5영업일 심사**를 거쳐야 합니다. "오늘 셋업 = 오늘 광고"가 아님을 미리 알리세요.

✅ **완료 기준**: 22자리 개발자 토큰 확보 (Test 등급). Basic 신청은 3단계 GCP 프로젝트 생성 후 제출.

> 💡 **신청 이후 — 브랜드 인증(Brand Verification): 필수 아님, "권장"(심사 단축)** (2026-07 기준)
>
> Basic 신청서를 제출하면 구글이 **접수 확인 메일**을 보냅니다
> (제목 예: *"We've received your Google Ads API Basic Access Application"*). 그 메일에
> **브랜드 인증을 완료하면 심사가 빨라진다(expedite)** 고 안내합니다.
> **필수는 아닙니다** — 메일에도 *"Otherwise, our team is currently processing your application"*
> (안 해도 팀이 그냥 심사 진행)이라 돼 있어, **안 해도 정상 심사(≈5영업일)는 그대로 진행**됩니다.
> **더 빨리 승인받고 싶을 때만** 하는 선택 작업입니다.
> 하기로 했다면 **자동화가 안 되고 사람이 직접** Google Cloud Console에서:
> 1. 개발자 토큰을 **GCP 프로젝트에 연결(연동)**
> 2. **OAuth 동의화면(Google 인증 플랫폼)** → 사용자 유형 **외부(External)** + 게시 상태를
>    **프로덕션(In production)** 으로 전환 (앱 게시)
> 3. **브랜딩 탭**에 앱 이름·로고·홈페이지·도메인 등 채우기
> 4. **"브랜딩 확인(Verify Branding)"** 클릭 → 완료 대기
> 5. 완료되면 **"브랜딩 게시(Publish branding)"** 클릭
>
> 📧 **모든 소통은 이메일로 옵니다.** 접수 확인, 추가 질문, 승인/거부 결과 모두 **신청서의 연락처
> 이메일**로 전달되니, **자주 확인하는 주소(가능하면 그룹/역할 주소)** 로 넣고 계속 확인하세요.
> 참고: 프로덕션으로 게시하면 앞서 말한 **리프레시 토큰 7일 만료 문제도 함께 해소**됩니다.
> 문서: https://developers.google.com/google-ads/api/docs/api-policy/brand-verification

### 3단계. GCP 결제 계정 (카드) — 없을 때만

GCP 프로젝트 생성과 API 활성화는 **Claude가 자동으로** 합니다. 사람이 할 건 결제 계정이
아예 없을 때 카드 등록뿐입니다.

1. https://console.cloud.google.com/billing → **"결제 계정 만들기"** (또는 무료 체험 가입 2단계)
2. 국가=대한민국, 통화=KRW, 이름·주소 입력
3. 신용/체크카드 등록 (신규는 보통 $300 무료 크레딧, Cloud Run 실제 청구는 거의 0)

> ⚠️ **실전 확인 — 결제 활성화에 최대 24시간** (2026-07 기준):
> 카드 등록을 마쳐도 결제 계정이 **바로 활성화되지 않을 수 있습니다.** gcloud로 보면
> `open: false` / `billingEnabled: false` 상태가 유지되고, 콘솔 상단에 **"무료 체험판을 사용하려면
> 선불 결제가 필요합니다. 이미 결제한 경우 24시간 이내에 계정에 반영됩니다"** 배너가 뜹니다.
> 확인 명령: `gcloud billing accounts list` (OPEN 열이 True가 되어야 함).
> - **결제 활성화 전에는 Cloud Run 배포가 막힙니다** → 활성화될 때까지(최대 24h) 기다렸다 배포.
> - 결제 계정은 **계정 단위**입니다. 새 GCP 프로젝트를 만들어도 **같은 결제 계정**을 물고 오므로,
>   프로젝트를 새로 만든다고 이 대기가 사라지지 않습니다.

✅ **완료 기준**: `gcloud billing accounts list`에서 결제 계정 **OPEN=True** (활성화 완료). 대기 중이면
배포만 미루고 나머지(OAuth·자격증명)는 계속 진행 가능.

### 4단계. OAuth 클라이언트 JSON 만들기

"누구의 권한으로 광고 계정에 접근하는가"를 정하는 인증 파일입니다.

> ⚠️ **실전 확인 — 콘솔 UI가 "Google 인증 플랫폼"으로 바뀜** (2026-07 기준). 예전 "OAuth 동의 화면"
> 단일 페이지 대신 4단계 마법사입니다. 아래 순서로 진행하세요.

**4-1. OAuth 동의(브랜딩) 구성** — https://console.cloud.google.com/auth/overview (프로젝트 선택 확인)
   - 마법사 4단계: **① 앱 정보**(앱 이름 + 지원 이메일) → **② 대상 = 외부(External)**
     (개인 Gmail은 "내부"가 비활성, 외부만 선택 가능) → **③ 연락처 이메일** → **④ 정책 동의 체크 → 만들기**

**4-2. 테스트 사용자 추가** — 좌측 **"대상"** 메뉴 → **테스트 사용자 → 사용자 추가** → **본인 이메일 입력 → 저장**
   > ⚠️ 이걸 빼먹으면 OAuth 로그인 때 **"액세스 차단됨(403)"**. 최다 실수. (게시 상태 "테스트 중"이면
   > 발급된 리프레시 토큰이 **7일 후 만료**될 수 있음 — 계속 쓰려면 나중에 "앱 게시"로 Production 전환하거나 재인증)

**4-3. OAuth 클라이언트 생성** — 좌측 **"클라이언트"** 메뉴 → **클라이언트 만들기**
   - 애플리케이션 유형 = **데스크톱 앱** → 이름 아무거나 → **만들기**
   - 생성 팝업에서 **JSON 다운로드** (윈도우: 파일 Shift+우클릭 → '경로로 복사')

✅ **완료 기준**: OAuth 클라이언트 JSON 파일(최상위 키 `installed`, client_id/client_secret 포함) 다운로드 완료.
설정 폴더에 `google_ads_oauth_client.json`으로 저장.

> 💡 **개념 정리** — *개발자 토큰 ≠ OAuth*. 개발자 토큰은 "이 앱이 Google Ads API를 써도 된다"는
> 허가증(1단계 MCC에서 발급), OAuth는 "누구의 계정 권한으로" 접근하느냐(2·4단계). **둘 다 있어야**
> 호출이 됩니다.

## Part 2 — Claude가 자동으로 하는 것 (원커맨드)

여기부터는 대부분 스크립트가 자동으로 합니다. 당신은 **값 붙여넣기**와 **브라우저 동의 1회**만 하면 됩니다.

### 5단계. 스킬 설치 확인

두 스킬 폴더가 `~/.claude/skills/` 아래에 있어야 합니다:
- `google-ads-direct-mcp` — 서버 본체(디멘드젠 툴 포함)
- `google-ads-demandgen-setup` — 셋업 자동화 스크립트

없으면 배포용 저장소에서 두 폴더를 `~/.claude/skills/` 로 복사하세요.

### 6단계. 처음부터 마법사로 시작 (계정 값 채우기)

Claude Code에게 이렇게 말하면 됩니다:

> **"디멘드젠 광고 설정 환경을 셋업해줘"**

그러면 `google-ads-demandgen-setup` 스킬이 실행되고, 아래 순서로 진행됩니다. 직접 명령으로도 가능:

```bash
# from-zero 마법사: 자동 가능한 건 자동, 웹 전용은 안내+검증
python scripts/guided_setup.py
```

마법사가 하는 일:
- **1) MCC / 2) 개발자 토큰**: Part 1에서 받은 **MCC ID(10자리)·토큰(22자)** 을 붙여넣으면 검증 후 `.env`에 저장
- **3) GCP 프로젝트 + 결제 + Ads API 활성화**: gcloud로 **완전 자동**
- **4) OAuth 클라이언트 JSON**: Part 1에서 받은 파일 경로를 붙여넣으면 설정 폴더로 자동 복사

설정·비밀값은 모두 `~/.google-ads-mcp/` 폴더에 저장됩니다.

### 7단계. 진단 (무엇이 준비됐는지 확인)

```bash
python scripts/preflight.py
```

Python·gcloud·로그인·결제·설정 파일 상태를 OK/누락으로 보여줍니다. `fail` 항목이 있으면
그 항목부터 해결하세요. 자주 걸리는 것: **개발자 토큰 Test 등급 / GCP 결제 미연결 / OAuth JSON 미저장**.

### 8단계. 원커맨드 배포

```bash
python scripts/bootstrap.py          # 확인하며 진행
python scripts/bootstrap.py --yes    # gcloud 설치 등 자동 승인
```

내부적으로 **9단계**를 자동 진행합니다: gcloud 설치 → 진단 → 의존성 설치 → **OAuth 동의(브라우저 1회)**
→ Cloud Run 배포 → Claude Code 등록 → 검증. 각 단계 실패 시 원인·해결을 자동 출력합니다.

- **당신이 할 일은 딱 하나**: OAuth 동의 화면이 브라우저(또는 출력된 URL)로 뜨면, 본인 구글 계정으로
  **"허용"** 클릭. (이때 4단계에서 테스트 사용자로 넣은 그 계정으로 로그인)
- 리전 변경: `--region us-central1` (기본값은 서울 `asia-northeast3`)
- 리프레시 토큰 새로 받기: `--reauth`

완료되면 **MCP URL** 과 **베어러 토큰**이 `~/.google-ads-mcp/connection.json` 에 저장됩니다.

> 🔐 **베어러 토큰 = 광고비 권한.** 이 토큰을 가진 사람은 연결된 광고 계정을 조작할 수 있습니다.
> 안전한 채널로만 공유하고, 절대 커밋/업로드하지 마세요.

### 9단계. 디멘드젠 준비 검증

```bash
python scripts/verify_demandgen.py
```

인증·읽기(계정 목록) + 디멘드젠 툴 존재 + **validate_only 쓰기 확인**(실제 생성·과금 없음)을 거쳐
**"디멘드젠 광고 설정 준비 완료 ✅"** 를 확인합니다.

또는 Claude Code에서 직접: **"내 구글애즈 계정 목록 보여줘"** → 계정 ID가 나오면 연결 성공.

## Part 3 — 실제 디멘드젠 광고 만들기 (셋업 이후)

셋업이 끝나면 Claude Code와 **대화만으로** 광고를 조립합니다. 모든 객체는 기본 **PAUSED(일시중지)** 로
만들어지므로 안전하게 조립하고, 마지막에 눈으로 확인한 뒤 켭니다.

### 미리 준비할 소재
- **이미지 URL** (공개 URL이어야 함 — 서버가 다운로드):
  - 가로 이미지 1.91:1 (1200×628 권장) 1개 이상
  - 정사각 이미지 1:1 (1200×1200 권장) 1개 이상
  - 로고 이미지 1개 이상
- **문구**: 헤드라인(최대 5개), 설명문(최대 5개), 업체명, 랜딩페이지 URL
- (영상 광고면) **YouTube에 먼저 업로드**한 영상 ID

### 조립 순서 (Claude와 대화)
1. "예산 만들어줘: 하루 3만원" → `create_campaign_budget`
2. "그 예산으로 디멘드젠 캠페인 생성" → `create_demand_gen_campaign` (PAUSED)
3. "광고그룹 만들어줘" → `create_demand_gen_ad_group` (PAUSED)
4. "이미지 애셋 업로드" → `upload_image_asset` / `upload_logo_asset` (이미지 URL 전달)
5. "이미지 광고 만들어줘" → `create_demand_gen_multi_asset_ad` (PAUSED, 애셋·문구·업체명·URL)
6. "정책 심사 상태 확인" → `validate_created_demand_gen_ad` (승인/거부/심사중 확인)
7. UI에서 확인 후 활성화 → `update_campaign` (status='ENABLED', `confirm_enable='ENABLE'`)

### 디멘드젠 API 제약 (꼭 알아둘 것)
- **입찰은 전환/클릭 기반** — 수동 CPC 불가. 전환추적이 없으면 `MAXIMIZE_CLICKS` 사용.
- **영상은 YouTube 먼저** — 생 영상 업로드 불가, YouTube video ID로 참조.
- **이미지는 URL로** — 모델에 base64로 넣지 말고 공개 URL 전달 (서버가 받아서 애셋 생성, ≤10MB).
- **행동유도문(CTA)** 은 표시 문구로: `"Learn more"`, `"Shop now"`, `"Sign up"` 등. `"LEARN_MORE"`
  같은 enum 토큰은 거부됨.

## Part 4 — 다른 PC에서 같은 서버 쓰기 (재배포 없음)

한 번 배포한 서버는 여러 PC에서 공유합니다. 새 PC에선 **URL + 베어러 토큰**만 있으면 됩니다.

```bash
python scripts/connect_new_pc.py                                  # connection.json 자동 사용
python scripts/connect_new_pc.py --url <…/mcp> --token <베어러>    # 직접 전달
```

`claude mcp add` 자동 실행 + `/healthz` 상태 확인까지 합니다. 서버를 다시 배포하지 않습니다.

수동으로 등록하려면:
```bash
claude mcp add --transport http google_ads_direct https://<your-url>/mcp \
  --header "Authorization: Bearer <YOUR_BEARER_TOKEN>"
claude mcp list
```

## 자주 막히는 곳 (트러블슈팅)

| 증상 | 원인 | 해결 |
|------|------|------|
| `DEVELOPER_TOKEN_NOT_APPROVED` | 토큰이 아직 Test 등급 | API Center → "기본 액세스 신청" → **영어 신청서 폼**(GCP 프로젝트ID + **설계문서** + 상세설명 필수) 제출 → **최대 5영업일** 심사. 승인 전엔 테스트 계정만 호출 가능 |
| GCP 결제가 `open:false` / 배포가 결제 오류로 막힘 | 신규 무료체험 결제 계정 활성화 지연 | 카드 등록 후에도 **최대 24시간** 걸림. `gcloud billing accounts list`에서 OPEN=True 될 때까지 대기 후 배포. 새 프로젝트 만들어도 해결 안 됨(결제는 계정 단위) |
| gcloud가 다른 계정으로 로그인돼 있음 | 이전에 다른 구글 계정으로 gcloud 사용 | `gcloud auth login <이메일>` (브라우저 열리는 기본 방식). `--no-launch-browser`는 코드 붙여넣기가 필요해 이 환경에선 부적합 |
| 광고계정(하위계정) 생성 시 "본인 인증" 후 "다시 시도" 반복 | 인증 팝업이 일부 브라우저에서 안 열림 | **실제 크롬**에서 MCC → 계정 → "새 계정 만들기"로 생성하면 인증이 정상 작동 |
| `invalid_grant` / 인증 오류 | 리프레시 토큰 만료·취소 | `bootstrap.py --reauth` 로 OAuth 재실행. `oauth_setup.py`는 **`python -u`** 로 실행해야 인증 URL이 바로 보임. OAuth 앱이 "테스트 중"이면 리프레시 토큰이 **7일 만에 만료** → 앱을 "게시(Production)"하거나 재인증 |
| OAuth 동의 시 "앱이 확인되지 않았습니다" | 앱이 Testing 상태(정상) | **고급 → "(안전하지 않음)으로 이동"** 클릭. 테스트 사용자로 등록된 본인 계정이면 안전 |
| `USER_PERMISSION_DENIED` / 특정 계정만 실패 | 로그인 계정과 대상 계정 관계 문제 | **전 단계를 같은 계정으로** 했는지 확인. MCC ID를 `GOOGLE_ADS_LOGIN_CUSTOMER_ID`에 (숫자만) 설정. 대상 계정이 그 MCC 밑에 연결됐는지 확인 |
| "액세스 차단됨(403)" (OAuth 로그인 시) | OAuth 동의화면 test users에 내 이메일 누락 | 4단계 2번의 **테스트 사용자에 본인 이메일 추가** |
| 배포 후 컨테이너가 바로 죽음 (secretmanager permission) | Cloud Run 런타임 SA에 시크릿 접근 권한 없음 | `bootstrap.py`가 자동 처리하지만, 안 되면 각 시크릿에 `roles/secretmanager.secretAccessor` 부여 후 재배포 |
| 캠페인 생성 시 "required field was not present" | v17+ 는 **EU 정치광고 표시** 필수 | 서버 캠페인 툴엔 이미 포함됨. 커스텀 캠페인 툴을 새로 만들면 `contains_eu_political_advertising` 필드 추가 |
| 디멘드젠 "Invalid call to action text" | CTA에 enum 토큰 넣음 | `"Learn more"` 같은 **표시 문구**로. 생략도 가능 |
